# AI 문제해결 프로젝트 후보 1순위–5순위

## 전제 조건

- **팀 구성**
  - 영화감독
  - 사업가
  - 마술사
  - 무용안무가
  - 피디
  - 기술감독

- **기간**: 6개월
- **방향성**
  - 대중성이 있어야 함
  - 가능한 한 새로운 AI 기술을 활용하면 좋음
  - 비개발자 구성원도 실제 제작에 기여할 수 있어야 함
  - 단순한 앱보다 콘텐츠·공연·영상·체험이 결합된 형태가 유리함

---

# 1순위. AI 고민 극장

## 한 줄 설명

관객이 자신의 고민이나 사회적 문제를 입력하면, AI가 그것을 영화적 장면, 마술적 장치, 무용적 움직임, 영상 이미지로 변환하고, 관객의 선택에 따라 다른 해결 루트를 보여주는 인터랙티브 AI 공연/영상 플랫폼.

## 핵심 콘셉트

사용자가 문제를 입력한다.

예시:

> 나는 창작을 계속하고 싶은데 돈이 안 돼서 불안해.

AI는 이 문제를 다음처럼 분석한다.

```text
감정: 불안, 갈망
갈등: 생계 vs 창작
상징: 꺼져가는 조명, 빈 객석, 찢어진 계약서
해결 방향: 작은 수익모델 실험, 공동 작업, 관객 검증
```

그 후 AI가 다음 결과를 생성한다.

```text
1. 1분짜리 시놉시스
2. 무대 연출안
3. 마술 연출 아이디어
4. 무용 동작 키워드
5. 영상 프롬프트
6. 관객에게 던질 질문
```

## 왜 1순위인가

- 팀원 6명의 역할이 가장 균형 있게 살아남
- 대중적인 주제인 “고민”, “불안”, “선택”, “문제 해결”을 다룰 수 있음
- AI 영상, 이미지, 음악, 텍스트 생성 기술을 자연스럽게 연결할 수 있음
- 웹 체험, 오프라인 쇼케이스, 전시, 공연, 피칭 모두 가능
- 6개월 안에 MVP 제작 가능성이 높음

## 역할 분배

| 역할 | 담당 영역 |
|---|---|
| 영화감독 | 이야기 구조, 장면 연출, 영상 톤 |
| 사업가 | 타깃 설정, 수익모델, 제휴, 피칭 |
| 마술사 | 관객 참여 방식, 트릭 구조, 놀라움 설계 |
| 무용안무가 | 감정별 움직임, 신체 표현, 퍼포먼스 설계 |
| 피디 | 제작 일정, 촬영, 쇼케이스 운영 |
| 기술감독 | AI 파이프라인, 웹앱, 데이터 구조, 프로토타입 |

## MVP 형태

- 사용자가 고민을 입력한다.
- AI가 문제를 감정, 갈등, 상징, 해결 방향으로 분해한다.
- AI가 짧은 장면, 안무, 마술 장치, 영상 프롬프트를 생성한다.
- 사용자는 2~3개의 선택지 중 하나를 고른다.
- 선택에 따라 다른 결과 장면이 나온다.

## 6개월 목표

```text
대표 문제 3개
각 문제별 선택지 3개
총 9개 결과 루트
3~5분짜리 체험형 쇼케이스
웹 데모 + 오프라인 발표 가능
```

## 리스크

- 실시간 영상 생성까지 욕심내면 기술 난이도가 높아짐
- 초반에는 “완전 실시간 생성”보다 “AI 생성 + 사전 제작된 결과 조합” 방식이 현실적임
- 연출 품질이 낮으면 단순 챗봇처럼 보일 수 있음

## 추천 형태

**웹 체험 + 오프라인 쇼케이스 결합형**

# 2순위. AI 마술 상담소

## 한 줄 설명

관객이 고민을 입력하면 AI가 그 고민을 상징하는 카드, 오브젝트, 이야기, 예언 문장을 만들고, 마술사가 이를 실제 마술처럼 보여주는 관객 참여형 AI 마술 체험.

## 핵심 콘셉트

사용자가 고민을 입력한다.

예시:

> 나는 중요한 선택을 앞두고 있는데 확신이 없어.

AI는 고민을 상징으로 변환한다.

```text
상징: 두 개의 문, 멈춘 시계, 접힌 지도
마술 장치: 예언 카드, 봉투, 선택된 오브젝트
결과 메시지: 선택은 정답을 찾는 일이 아니라 기준을 드러내는 일이다.
```

마술사는 AI가 생성한 상징과 메시지를 바탕으로 관객 앞에서 마술을 진행한다.

## 왜 2순위인가

- 기술 난이도가 상대적으로 낮음
- 관객 반응을 얻기 쉬움
- 마술사의 전문성이 강하게 살아남
- AI가 “예언”, “상징”, “해석”과 잘 어울림
- 오프라인 팝업, 행사, 전시, 브랜드 이벤트로 확장 가능

## 역할 분배

| 역할 | 담당 영역 |
|---|---|
| 영화감독 | 전체 분위기, 장면 구성 |
| 사업가 | 팝업, 이벤트, 브랜드 제휴 |
| 마술사 | 핵심 체험, 트릭 설계, 관객 인터랙션 |
| 무용안무가 | 일부 감정 동작 또는 오프닝 퍼포먼스 |
| 피디 | 현장 운영, 제작 관리 |
| 기술감독 | 입력 시스템, AI 해석, 결과 출력 화면 |

## MVP 형태

- 관객이 태블릿이나 웹페이지에 고민을 입력한다.
- AI가 고민을 상징 카드 3장으로 변환한다.
- 관객이 카드나 오브젝트를 선택한다.
- 마술사는 선택 결과를 예언한 것처럼 보여준다.
- 마지막에 AI가 짧은 해석 메시지를 제공한다.

## 6개월 목표

```text
고민 카테고리 5개
상징 카드 30개
마술 루틴 3개
관객 1명당 3~5분 체험
소규모 오프라인 쇼케이스 가능
```

## 리스크

- AI 기술의 새로움이 약해 보일 수 있음
- 마술 연출이 약하면 상담 챗봇처럼 보일 수 있음
- 무용안무가의 역할이 상대적으로 작아질 수 있음

## 추천 형태

**오프라인 체험형 팝업 / 소규모 공연형 쇼케이스**

# 3순위. AI 문제해결 게임룸

## 한 줄 설명

사용자의 문제를 작은 방 하나로 시각화하고, 방 안의 오브젝트를 클릭하면 문제 해결 단서가 나오는 인터랙티브 웹 경험.

## 핵심 콘셉트

사용자가 문제를 입력한다.

예시:

> 사람들 앞에서 말하는 게 무섭다.

AI가 문제를 하나의 방으로 바꾼다.

```text
방의 분위기: 어둡고 좁은 발표 대기실
오브젝트: 꺼진 마이크, 닫힌 커튼, 깨진 거울, 작은 조명, 잠긴 문
상징 감정: 긴장, 자기검열, 회피
```

사용자는 방 안의 오브젝트를 클릭한다.

```text
마이크: 말을 잘하는 것보다 한 문장을 끝내는 연습이 먼저다.
거울: 남의 시선보다 자신의 녹음 피드백을 먼저 보라.
문: 첫 발표는 많은 사람 앞이 아니라 3명 앞에서 시작하라.
```

## 왜 3순위인가

- 웹앱 MVP로 만들기 좋음
- 사용자가 즉시 이해하기 쉬움
- 게임, 상담, AI 이미지 생성, 인터랙티브 UI를 결합할 수 있음
- 대중적인 체험형 서비스로 확장 가능
- 추후 모바일 앱이나 전시 설치물로 발전 가능

## 역할 분배

| 역할 | 담당 영역 |
|---|---|
| 영화감독 | 방의 장면 구성, 분위기 연출 |
| 사업가 | 서비스 모델, 유료화, 사용자 분석 |
| 마술사 | 오브젝트 클릭 시 반전/놀라움 구조 |
| 무용안무가 | 감정별 움직이는 오브젝트 또는 캐릭터 모션 |
| 피디 | 콘텐츠 제작 관리, 테스트 운영 |
| 기술감독 | 웹앱, AI 생성, 인터랙션 구현 |

## MVP 형태

- 사용자가 고민을 입력한다.
- AI가 고민을 방의 콘셉트로 변환한다.
- AI가 방 안의 오브젝트 5개를 생성한다.
- 각 오브젝트는 해결 단서나 행동 제안을 가진다.
- 최종적으로 사용자는 자신의 문제 해결 리포트를 받는다.

## 6개월 목표

```text
문제 카테고리 5개
방 템플릿 10개
오브젝트 템플릿 50개
웹 기반 인터랙티브 데모
개인 결과 저장 기능
```

## 리스크

- 공연자들의 역할이 약해질 수 있음
- 기술감독의 개발 부담이 커질 수 있음
- 이미지 생성 결과의 일관성을 관리해야 함

## 추천 형태

**웹앱 MVP / 전시형 인터랙티브 설치물**

# 4순위. AI 감정 안무 생성기

## 한 줄 설명

사용자가 감정을 입력하면 AI가 그 감정을 해석해 무용 동작, 음악, 조명, 영상 배경, 장면 구성을 생성하는 감정 기반 퍼포먼스 시스템.

## 핵심 콘셉트

사용자가 감정을 입력한다.

예시:

> 불안하지만 앞으로 가고 싶은 감정.

AI는 이를 퍼포먼스 요소로 변환한다.

```text
움직임: 수축 → 떨림 → 정지 → 작은 전진 → 확장
음악: 낮은 드론 사운드에서 점진적인 리듬으로 변화
조명: 차가운 청색에서 따뜻한 백색으로 이동
영상: 좁은 방에서 열린 들판으로 전환
```

## 왜 4순위인가

- 무용안무가의 전문성이 가장 강하게 살아남
- AI와 신체 표현을 결합하는 점에서 예술적 차별성이 있음
- 공연, 전시, 미디어아트로 발전 가능
- 감정 데이터와 움직임 데이터를 쌓을 수 있음

## 역할 분배

| 역할 | 담당 영역 |
|---|---|
| 영화감독 | 영상 배경, 장면 흐름, 감정선 |
| 사업가 | 공연/전시/교육 프로그램화 |
| 마술사 | 퍼포먼스 중 전환 장치, 관객 참여 요소 |
| 무용안무가 | 핵심 안무 체계, 감정별 움직임 사전 |
| 피디 | 공연 제작, 리허설, 촬영 관리 |
| 기술감독 | AI 분석, 모션 데이터, 영상/음악 생성 연결 |

## MVP 형태

- 사용자가 감정 문장을 입력한다.
- AI가 감정을 움직임 키워드로 변환한다.
- 안무가는 이를 바탕으로 짧은 안무를 만든다.
- AI가 조명, 음악, 영상 배경 프롬프트를 생성한다.
- 최종적으로 1~2분짜리 감정 퍼포먼스를 만든다.

## 6개월 목표

```text
감정 카테고리 8개
감정별 움직임 사전 제작
1~2분 퍼포먼스 3개
AI 생성 영상 배경 3종
쇼케이스 공연 가능
```

## 리스크

- 대중적 설명이 부족하면 순수 예술 프로젝트처럼 보일 수 있음
- “문제 해결”보다는 “감정 표현”에 가까워질 수 있음
- 사업화 방향이 상대적으로 좁을 수 있음

## 추천 형태

**미디어아트 공연 / 감정 기반 AI 퍼포먼스 쇼케이스**

# 5순위. AI 사회문제 예고편 생성기

## 한 줄 설명

사용자가 사회문제를 선택하면 AI가 그 문제를 바탕으로 가상의 영화 예고편, 포스터, 로그라인, 캠페인 문구를 생성하는 AI 콘텐츠 제작 시스템.

## 핵심 콘셉트

사용자가 사회문제를 선택한다.

예시:

```text
AI 실업
청년 고립
지방 소멸
기후 불안
고령화
학교폭력
```

AI는 이를 영화 예고편 기획안으로 바꾼다.

```text
제목: 마지막 출근
장르: 근미래 드라마
로그라인: AI에게 업무를 넘긴 한 청년이, 자신의 쓸모를 다시 정의하기 위해 마지막 프로젝트를 시작한다.
포스터 문구: 사라지는 것은 일이 아니라, 일의 의미일지도 모른다.
예고편 구성: 도시 전경 → 빈 사무실 → 자동화 시스템 → 주인공의 선택
```

## 왜 5순위인가

- 영화감독과 피디의 전문성이 강하게 살아남
- 사회문제를 대중적인 영화 문법으로 전달할 수 있음
- 기관, 캠페인, 교육 콘텐츠와 연결 가능
- 숏폼 콘텐츠로 확산하기 쉬움

## 역할 분배

| 역할 | 담당 영역 |
|---|---|
| 영화감독 | 예고편 구조, 장면 연출, 영화 문법 |
| 사업가 | 기관 제휴, 캠페인화, B2B 제안 |
| 마술사 | 예고편 속 반전 구조, 관객 참여 장치 |
| 무용안무가 | 상징적 움직임, 군무, 신체 이미지 |
| 피디 | 제작 진행, 영상 소스 관리, 촬영 |
| 기술감독 | 생성 파이프라인, 웹 입력 시스템, 영상 도구 연결 |

## MVP 형태

- 사용자가 사회문제를 고른다.
- AI가 로그라인, 주인공, 장면 구성, 포스터 문구를 생성한다.
- AI 이미지/영상 도구로 포스터와 짧은 예고편 이미지를 만든다.
- 최종적으로 30초짜리 예고편 콘셉트 또는 영상 데모를 만든다.

## 6개월 목표

```text
사회문제 5개
문제별 예고편 기획안 5개
포스터 5종
30초 예고편 데모 1~2개
기관 피칭용 패키지 제작
```

## 리스크

- 기존 AI 영상 생성 사례와 차별화가 약할 수 있음
- 마술사와 무용안무가의 역할이 제한적일 수 있음
- 결과물이 단순한 “AI 단편 예고편”처럼 보일 수 있음

## 추천 형태

**사회문제 캠페인용 AI 영상 제작 툴 / 피칭용 데모**

# 최종 결론

가장 추천하는 방향은 **AI 고민 극장**이다.

이 프로젝트는 영화, 사업, 마술, 무용, 제작, 기술이 모두 자연스럽게 결합된다. 또한 관객이 직접 문제를 입력하고 결과를 체험하는 구조이기 때문에 대중성도 확보하기 쉽다.

초기 MVP는 완전한 실시간 영상 생성보다 다음 방식이 현실적이다.

```text
사용자 입력
→ AI 문제 분석
→ 장면/마술/안무/영상 프롬프트 생성
→ 사전 제작된 리소스와 AI 생성 결과 조합
→ 관객 선택에 따른 분기형 결과 제공
```

6개월 안에 목표로 삼을 최종 결과물은 다음 정도가 적절하다.

```text
웹 기반 체험 데모
대표 문제 3개
문제별 선택지 3개
총 9개 결과 루트
3~5분짜리 오프라인 쇼케이스
기관/브랜드 제휴용 피칭 자료
```
